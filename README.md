# peptidly-website

Public marketing and legal site for the [Peptidly](https://peptidly.co.uk) Android app.
Hosted on GitHub Pages, served from `peptidly.co.uk`.

This repo intentionally has **zero dependencies** — it's plain HTML, CSS, and one
build script that uses only built-in Node modules. Pushing to `main` deploys via
GitHub Pages with no build step on the GitHub side.

## Layout

```
peptidly-website/
├── README.md                       ← you are here
├── CNAME                           ← peptidly.co.uk (tells GitHub Pages the domain)
├── .gitignore
├── index.html                      ← landing page (hand-written)
├── style.css                       ← all styles, single file (~150 lines)
├── privacy.html                    ← generated by build script
├── terms.html                      ← generated by build script
├── health-disclaimer.html          ← generated by build script
└── scripts/
    └── build-from-content.mjs      ← reads ../Peptides/helix/legal/content.ts → renders the three .html files
```

## Updating the legal text

The legal documents are **derived** from `helix/legal/content.ts` in the
sibling [Peptides repo](https://github.com/<your-username>/Peptides). Editing
the `.html` files directly will get overwritten on the next build.

To update:

```bash
# 1. Edit the constants in the app repo
cd C:\Users\norbe\source\repos\Peptides\helix
# ... edit legal/content.ts ...

# 2. Re-run the build script in the website repo
cd C:\Users\norbe\source\repos\peptidly-website
node scripts/build-from-content.mjs

# 3. Commit and push the website repo
git add privacy.html terms.html health-disclaimer.html
git commit -m "update legal text"
git push
```

GitHub Pages picks up the new commit within a minute.

## Updating other pages

`index.html` is hand-written. Edit it directly if the landing copy changes.

`style.css` controls all visual styling. Single file, no preprocessor.

## First-time deployment

If this repo doesn't have a remote yet, here's the full setup:

1. **Create a public GitHub repo** named `peptidly-website` under your account.
   Do NOT initialize with a README from the GitHub UI — we'll push our own.

2. **Push the local files:**
   ```bash
   cd C:\Users\norbe\source\repos\peptidly-website
   git init
   git add .
   git commit -m "initial commit"
   git branch -M main
   git remote add origin https://github.com/<your-username>/peptidly-website.git
   git push -u origin main
   ```

3. **Enable GitHub Pages:**
   - Repo → Settings → Pages
   - Source: **Deploy from a branch**
   - Branch: **main**, folder: **/** (root)
   - Save

4. **Verify the temporary URL works:** `https://<your-username>.github.io/peptidly-website/`

5. **Configure custom domain** (Settings → Pages → Custom domain): `peptidly.co.uk`

6. **Configure DNS at Namecheap:**
   - Domain List → Manage `peptidly.co.uk` → Advanced DNS
   - Delete any default records
   - Add four A records for the apex (`@`):
     ```
     @  A  185.199.108.153
     @  A  185.199.109.153
     @  A  185.199.110.153
     @  A  185.199.111.153
     ```
   - Add CNAME for `www` → `<your-username>.github.io`
   - Save and wait for DNS propagation (5 min to a few hours)

7. **Wait for HTTPS** to provision in Settings → Pages, then check **Enforce HTTPS**.

8. **Verify:** `https://peptidly.co.uk/privacy` should load.

## Notes

- The site has no JavaScript, no analytics, no cookies, no fonts loaded from
  external CDNs. Pure HTML + CSS + the system font stack. Page weight is
  under 10 KB total.
- The text in the legal documents is **not legal advice**. The drafts were
  written based on what the Peptidly app actually does, but a real lawyer
  should review them before any major distribution event.
